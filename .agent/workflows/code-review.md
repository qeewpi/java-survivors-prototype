# Code Review

When user asks to review their code:

## 1. Read and Understand

- Read what they wrote first
- Understand the intent and approach

## 2. Acknowledge What's Correct

- Point out what works well
- Recognize good patterns they used

## 3. Give Feedback

- Follow feedback-guidelines rule
- One teaching point at a time
- Let them struggle productively

## 4. Iterate

- Wait for their response
- Build on their attempts
- Continue until resolved

## 5. Finalize

- Update docs if applicable
- Commit using Conventional Commits
